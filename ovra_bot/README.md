# Ovra PM-Bot 🤖

Telegram-бот для автоматического поиска, парсинга и трекинга задач из рабочих чатов. Бот анализирует переписку команды, формирует карточки задач с помощью ИИ (OpenRouter) и работает в связке с бэкендом [Ovra](https://github.com/Egooroh/Ovra.git) для автоматической публикации карточек в YouGile.

## 🌟 Основные возможности
* **Автоматический поиск задач:** Бот читает чат и по ключевым словам (эвристике) понимает, что обсуждается задача.
* **Ручное создание:** Любое сообщение в чате можно превратить в задачу, просто поставив на него реакцию (✍️ или 🔥).
* **AI-парсинг:** ИИ автоматически извлекает из текста название задачи, дедлайн, исполнителя и описание.
* **Система аппрува:** Задачи не создаются бездумно. Бот присылает сформированную карточку Project Manager-у в личные сообщения для подтверждения.
* **Умный кэш:** Бот запоминает контекст и работает через защищенный SOCKS5/HTTP прокси.

---

## 📌 Требования перед запуском
Для корректной работы бота на вашем компьютере должны быть выполнены следующие условия:
1. Запущен локальный сервер на Go — [Ovra Backend](https://github.com/Egooroh/Ovra.git) (по умолчанию на `localhost:8080`).
2. Включен VPN или локальный прокси-клиент (например, V2rayTun) для доступа к API Telegram и OpenRouter.
3. Установлен **Node.js** (версии 18+).

## 🛠 Установка и настройка

### 1. Установка зависимостей
Откройте терминал в папке с проектом и выполните команду для скачивания всех нужных модулей:
```bash
npm install
```

### 2. Настройка окружения (.env)
Создайте в корне проекта файл `.env` (если его нет) и заполните его по следующему шаблону:
```env
TELEGRAM_BOT_TOKEN="ваш_токен_от_BotFather"
OPENROUTER_API_KEY="ваш_ключ_от_OpenRouter"
BACKEND_URL="http://localhost:8080"
AI_MODEL="qwen/qwen3.7-plus" # или любая другая модель, например mistralai/mistral-7b-instruct:free
PROXY_URL="[http://127.0.0.1:10801](http://127.0.0.1:10801)" # Ваш локальный прокси

# Оставьте пустым, бот заполнит это автоматически при команде /start
PM_CHAT_ID="" 
```

---

## 🚀 Запуск бота
Когда зависимости установлены и Go-сервер работает, введите в терминале:
```bash
npm start
```
*Если всё настроено верно, бот сообщит об успешном запуске в консоли.*

## ⚙️ Инструкция по использованию в Telegram

### Шаг 1: Назначение PM-а (Менеджера)
Найдите вашего бота в Telegram (по юзернейму, который вы задали в BotFather) и напишите ему в **личные сообщения**:
```text
/start
```
Бот ответит: *«✅ Вы успешно назначены PM-ом!»*. Он автоматически запишет ваш Telegram ID в файл `.env`. Теперь все новые задачи из рабочих чатов будут приходить вам в личку на подтверждение.

### Шаг 2: Подключение к рабочему чату
1. Добавьте бота в вашу рабочую Telegram-группу команды.
2. **Обязательно выдайте боту права Администратора**. Без этого Telegram не позволит боту читать сообщения пользователей.

### Шаг 3: Как создавать задачи

**Способ А: Автоматически (Эвристика)**
Просто общайтесь в чате. Если кто-то напишет сообщение с явной постановкой задачи, бот сам её перехватит. 
*Пример:*
> «Ребята, нужно сделать новый экран профиля пользователя. Дедлайн до пятницы, задача на @rikby»

**Способ Б: Вручную (Через реакции)**
Если в сообщении нет явных слов-триггеров, но это баг или задача, поставьте на это сообщение в Telegram реакцию **✍️** или **🔥**. Бот мгновенно возьмет этот текст в работу.
*Пример:*
> «У нас не работает отправка медиа, если вес файла больше 10мб. Сервер падает.» -> *(Пользователь ставит реакцию ✍️)*

### Шаг 4: Подтверждение (Аппрув)
Как только бот поймает задачу (любым из способов), он пришлет её PM-у в личные сообщения.
* Нажмите **[✅ Одобрить]**, чтобы задача улетела в бэкенд Ovra и автоматически появилась на доске в YouGile.
* Нажмите **[❌ Отклонить]**, если бот сработал ложно или задача не нужна.

## 📊 Мониторинг статуса
Вы всегда можете проверить состояние системы. Напишите боту команду:
```text
/stats
```
Бот выдаст сводку о том, подключен ли прокси, отвечает ли Go-бэкенд, доступен ли ИИ и сколько сообщений сейчас находится в кэше.
